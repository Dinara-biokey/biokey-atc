🇰🇿 Билайн Казахстан — подключается напрямую (WebSocket встроен)
Облачная АТС Билайн Казахстан уже поддерживает WebSocket — промежуточный Asterisk не нужен.
Каждая компания получает свой субдомен вида COMPANY.ats.beeline.kg
Шаг 1. Войдите в личный кабинет АТС: https://COMPANY.ats.beeline.kg
Шаг 2. Перейдите в Сотрудники → выберите сотрудника → скопируйте его Логин (часть до @) и установите SIP-пароль.
Шаг 3. Заполните поля ниже:
WSS-сервер: wss://COMPANY.ats.beeline.kg
Домен: COMPANY.ats.beeline.kg
Логин: ivanov.ivan (имя сотрудника из ЛК)
Пароль: SIP-пароль из настроек сотрудника

Порты (разрешить на роутере/файрволе):
TCP: 80, 443, 8080, 5060, 5061
UDP: 5060, 10000–65535
Подсети: 91.247.60.32/28 и 91.247.60.80/29
⚠️ Важно: Билайн требует WebSocket-шлюза
Билайн Бизнес работает по стандартному SIP/UDP — браузер к нему напрямую подключиться не может.
Нужен промежуточный Asterisk-сервер, который принимает WebSocket от браузера и пересылает звонки в Билайн.
Схема:
Браузер (JsSIP/WSS) → Asterisk (ваш сервер) → SIP/UDP → sip.beeline.ru

Настройки Asterisk (sip.conf / pjsip.conf): 
; Транк к Билайн (исходящий)
[beeline-trunk]
type=peer
host=sip.beeline.ru
outboundproxy=msk.sip.beeline.ru
username=7XXXXXXXXXX        ; ваш номер Билайн
secret=ВАШ_ПАРОЛЬ
fromuser=7XXXXXXXXXX
fromdomain=sip.beeline.ru
insecure=port,invite
dtmfmode=rfc2833
disallow=all
allow=alaw&ulaw
directmedia=no

; Регистрация у Билайн
register => 7XXXXXXXXXX:ВАШ_ПАРОЛЬ@sip.beeline.ru/7XXXXXXXXXX

; Включить WebSocket для браузеров (http.conf)
; enabled=yes
; bindaddr=0.0.0.0
; bindport=8088
; tlsenable=yes
; tlsbindaddr=0.0.0.0:8089

В поле «WSS-сервер» ниже — укажи адрес твоего Asterisk-сервера (не Билайна):
wss://твой-asterisk.ru:8089/ws
Подключение
Как получить данные от Билайн
1. Подключите услугу «SIP-телефония» в личном кабинете Билайн Бизнес или через менеджера.
2. Получите от менеджера: Логин (SIP User ID), Пароль, SIP-сервер.
3. Стандартные данные: сервер sip.beeline.ru, домен sip.beeline.ru.
4. Настройте Asterisk как шлюз (схема выше) — укажите его WSS-адрес в поле выше.
5. В Asterisk каждый оператор — это внутренний SIP-аккаунт (extension), который регистрируется через WebSocket.
